# 从自建服务器到 GitHub Pages + Cloudflare + 微信云开发 — 迁移与排错全记录

本文档记录 **游戏墓地（网站）** 与 **被遗忘的游戏时光（微信小程序）** 从「依赖自有服务器」转向 **无服务器 / 托管平台** 的目标架构、实施步骤、已遇问题与处理方式。**仓库为公开仓库时，切勿提交密钥、Token、私钥或 `.env`。**（云开发 **环境 ID** 会出现在小程序端，属正常，不等同于密钥。）

---

## 一、架构对照

| 部分 | 原方案（概要） | 现方案 |
|------|----------------|--------|
| **网站** | VPS + Nginx，静态页 + Node `/api`（献花、留言、在线状态等） | **GitHub Actions → GitHub Pages** 纯静态；**Cloudflare** 做 DNS（可选代理） |
| **网站 API** | 同机 Node 进程（如 `127.0.0.1:8788`） | **推荐**：**Cloudflare Worker + KV**（`workers/api/`，路由 `域名/api/*`），与 Pages 同域名；不配 API 时前端会探测 `/api/health` 并关闭世界同步等，避免刷屏 |
| **小程序** | — | **微信云开发 CloudBase**：云数据库 `game_list` / `user_collect` + 6 个云函数 |

## 二、网站：GitHub Pages + Cloudflare

### 2.1 目标

- 自定义域名（如 `youximudi.com`）访问静态站点，**无需维护 VPS**。
- 仓库：静态资源为根目录 `index.html`、`css/`、`js/`、`data/`，以及根目录 **`CNAME`**（自定义域名主机名）。

### 2.2 GitHub 侧流程

1. **Settings → Pages → Build and deployment**  
   - **Source** 选择 **GitHub Actions**（不要选「从分支部署」与当前 workflow 混用导致状态混乱）。
2. 工作流文件：`.github/workflows/github-pages.yml`  
   - 打包 `_site`：`index.html`、`css`、`js`、`data`、`CNAME`。  
   - 使用 `actions/configure-pages@v5`、`actions/upload-pages-artifact@v3`、`actions/deploy-pages@v5`（与官方静态站模板对齐）。  
   - `concurrency`: **勿**对进行中的 Pages 部署 `cancel-in-progress: true`，避免半成品上线。  
   - 支持 **`workflow_dispatch`**，便于在 Actions 里手动 **Run workflow**。
3. **Settings → Pages → Custom domain**  
   - 填写根域名（如 `youximudi.com`），等待 **DNS check successful**。  
   - **Enforce HTTPS** 在证书就绪后可选；若长期不可用，多为 Cloudflare 代理或证书仍在申请中。
4. **首次部署与环境**  
   - 若 Pages 提示 *Workflow details will appear once your site has been deployed*，说明 **尚未有一次成功的 Actions 部署**。  
   - 打开 **Actions** 查看 **GitHub Pages** workflow 是否成功；若显示 **Waiting for approval**，到 **Settings → Environments → `github-pages`** 处理审批规则或批准本次部署。
5. **验证顺序**  
   - 先确认 **`https://<用户名>.github.io/<仓库名>/`** 能打开；  
   - 再确认 **`https://自定义域名/`** 正常（避免只改 DNS 未发版）。

### 2.3 Cloudflare DNS 建议

| 记录类型 | 名称 | 内容 | 说明 |
|----------|------|------|------|
| A | `@`（根域名） | 四条，分别为 `185.199.108.153`、`185.199.109.153`、`185.199.110.153`、`185.199.111.153` | [GitHub 文档：根域名](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site#configuring-an-apex-domain) |
| CNAME | `www` | `<用户名>.github.io` | 不要加 `https://` 与路径 |

**注意：**

- 控制台导入文件名 **仅支持 `.json` 或 `.csv`**（网站数据与此无关）；小程序数据库导入见下文。
- **不要在 Cloudflare 的同一区域里**再挂指向 **DNSPod** 的 **NS** 记录与 Cloudflare 权威解析混用，易解析异常。
- **仅使用 GitHub Pages 时**，根域名 **不要再指向原 VPS IP**（如 `154.198.x.x`），否则流量到不了 GitHub。
- **IP 必须写全**：例如 `185.199.111.153`，**不能**写成 `185.199.111.15`。  
- **HTTPS 证书与校验**：可先对根域名与 `www` 使用 **仅 DNS（灰云）**，待 GitHub 检测与访问稳定后，再按需改为橙云；橙云时 SSL/TLS 建议 **完全（Full）**，避免错误模式导致证书无法签发。

### 2.4 网站侧问题与处理（速查表）

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| **502**，响应头 `Server: cloudflare` | 源站不可达（曾指 VPS 但机器/反代挂了） | 改 DNS 到 GitHub 四条 A；或修复源站 |
| **Fastly error: unknown domain** | 域名已到 GitHub CDN，但自定义域名未在 GitHub 完成登记 / DNS 检测未完成 | 完成 Actions 部署；DNS 用灰云重试；必要时删掉自定义域名再保存触发重验 |
| **DNS Check in Progress**（长期） | 传播慢、记录错误、橙云干扰校验 | 核对四条 A；先灰云；等待或重配自定义域名 |
| **404 – There isn't a GitHub Pages site here** | 无成功部署或项目站点未发布 | 见 §2.2：Actions 成功、`github.io/仓库名` 先通 |
| **`GET /api/world-state` 404、`POST /api/*` 405**（自定义域名 + 橙云） | `/api/*` 未接到本仓库 Worker，或被旧 Worker/规则拦截 | 见 §2.6：部署 `workers/api`、路由 `域名/api/*`；删除只支持 GET 的旧 Worker |
| 导入数据库报 **JSON Lines** | 微信云数据库要求（易与网站混淆） | 见 **第三节** |

### 2.5 网站形态说明（设计如此）

首页为 **Canvas 像素墓园**（可走动、墓碑弹窗），数据来自 **`data/games.json`**，**不是**传统文章列表页。与微信小程序的信息架构不同，属产品原有设计。

### 2.6 Cloudflare Worker + KV（推荐，与 Pages 同域名）

- **代码**：`workers/api/src/index.js`，配置 `workers/api/wrangler.toml`（KV 绑定 `YOUXIMUDI_KV`）。  
- **部署**：本机 `npm run deploy`，或配置 Secrets 后由 **`.github/workflows/cloudflare-worker.yml`** 在推送 `workers/api/` 时自动部署（见 `workers/api/README.md`）。  
- **路由**：`youximudi.com/api/*`（按实际域名修改）；勿再保留「只支持 GET」的旧 Worker，否则 **POST 会 405**。  
- **数据**：KV 中键设计与自建 `server/api-core.mjs` 一致。若有历史 `server/data/kv.json`，可用 **`server/migrate-to-cloudflare-kv.mjs`** 写入 KV；反向导出仍用 **`server/migrate-from-cloudflare.mjs`**。  
- **橙云**：与 GitHub Pages 并存时，仅 `/api/*` 进 Worker，其余仍到 Pages 源站。

### 2.7 自建方案（可选，全功能 API 同机）

若不用 Worker、仍要 **Node `/api` 与静态同机**，按 `deploy/部署说明.txt` 与 `deploy/nginx-youximudi.example.conf` 自建 Nginx + Node；`.github/workflows/deploy-selfhosted.yml` 使用 **`secrets.*`**，**私钥与主机信息仅存 GitHub Secrets，勿写入仓库文件。** 此情形下 Cloudflare 上应 **不要** 让 `域名/api/*` 再指向 Worker，以免请求到不了本机。

## 三、小程序：微信云开发

### 3.1 工程位置

本仓库内 **`miniprogram/`** 目录为微信开发者工具打开的根（含 `project.config.json` 与 `cloudfunctions/`）。

### 3.2 配置要点

- **`miniprogram/miniprogram/envList.js`**：`envId` 为云开发环境 ID（会进客户端，**勿与密钥混淆**）。  
- **`project.config.json`**：`appid`、`miniprogramRoot`、`cloudfunctionRoot`。  
- **云函数**（均需上传并安装依赖）：`getGameList`、`getGameDetail`、`updateViewCount`、`getRandomGame`、`userCollectOperate`、`getUserCollectList`。

### 3.3 数据库

- 集合：**`game_list`**、**`user_collect`**。  
- 权限：见 `miniprogram/database/数据库权限说明.txt`。  
- **导入格式**：控制台要求 **JSON Lines**（一行一条 JSON），且上传文件名 **只能是 `.json` 或 `.csv`**。  
  - 使用 **`miniprogram/database/game_list.import.json`**（内容为 JSON Lines，扩展名为 `.json`）。  
  - **不要**使用带 `[` `]` 的 JSON 数组文件直接导入。  
- 示例数据条数建议 **≥10** 便于审核。

### 3.4 云函数依赖

各云函数目录下执行 **`npm install`**（或开发者工具「云端安装依赖」）。**公开仓库不要提交 `node_modules/`**（见仓库根 `.gitignore`）。

### 3.5 小程序问题与处理（速查表）

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| **`Error: timeout`**（`WAServiceMainContext`） | 灰度基础库 + 模拟器；或云函数未部署/网络 | `project.private.config.json` 中 **`libVersion` 使用稳定版**（如 `3.4.10`），勿长期用灰度 **3.15.x**；关闭 **`app.json` 的 `lazyCodeLoading`**；`utils/cloud.js` 已为 `callFunction` 设置较长 **timeout** |
| 启动 **Launch Time** 数秒 | 同上 | 同上；真机对比模拟器 |
| 云函数失败 | 未上传、环境 ID 错误、权限 | 控制台云端测试；核对 `envList` 与集合权限 |

## 四、公开仓库安全清单（必查）

- **不要提交**：`.env`、`*_rsa`、`*_ed25519`、`*.pem`、含 **Token/密码** 的笔记、**SSH 私钥全文**。  
- **GitHub Actions**：敏感信息只放在仓库 **Settings → Secrets and variables**，workflow 里用 **`${{ secrets.XXX }}`**，**不要**写死。  
- **自建部署脚本**（如 `deploy-selfhosted.yml`）中的 `DEPLOY_SSH_KEY` 等 **仅存在于 Secrets**。  
- 小程序 **云环境 ID**、小程序 **AppID** 会出现在客户端或配置中，属常见公开信息，**仍不要**把云开发 **管理员密钥**、**API 密钥** 放进仓库。
